# Phase 32-04 PLAN — Live validation, deploy, commit, push and learnings

## Objetivo

Validar a Phase 32 ponta a ponta em ambiente real sem quebrar o channel 5, fazer build com limite de CPU, documentar evidencias, commitar e pushar somente depois de tudo verde.

## Preflight

1. Confirmar worktree e separar mudancas preexistentes.
2. Criar backup SQL do channel 5 antes de qualquer escrita live.
3. Confirmar token de operador via Vault, sem imprimir segredos.
4. Confirmar rota atual de producao e rollback.
5. Confirmar que o hotfix temporario ainda funciona antes de regenerar.

## Validacao Live

1. Iniciar regeneration para channel 5.
2. Abrir authorize URL em Brave/Chrome logado.
3. Capturar URL final `localhost:1455/auth/callback` via UI ou Chrome DevTools, sem imprimir tokens.
4. Completar regeneration no backend.
5. Validar metadata:
   - `authenticated=true`
   - `has_refresh_token=true`
   - `requires_regeneration=false`
   - `expires_at` futuro
6. Rodar refresh manual e confirmar novo `last_refresh`.
7. Rodar probe upstream e confirmar `Probe OK`.
8. Rodar smokes:
   - local `/v1/models`
   - local `/v1/chat/completions` non-stream
   - local `/v1/chat/completions` stream
   - local `/v1/responses` stream
   - publico `https://router.atius.com.br/v1/chat/completions`
9. Validar negativo:
   - API key interna invalida retorna auth interna do Router.
   - upstream auth invalida controlada retorna `codex_upstream_token_invalidated` ou equivalente planejado.

## Deploy

- Usar somente fluxo runtime protegido do projeto.
- Se houver rebuild/deploy, aplicar CPU guard via wrappers.
- Reiniciar `systemctl --user restart container-router-ai-atius.service` apenas apos build/test verde e backup.
- Validar health apos restart.

## Fechamento

1. Atualizar SUMMARY(s) da Phase 32 com evidencias.
2. Rodar `$gsd-extract-learnings 32` apos existir SUMMARY.
3. Atualizar Obsidian/GBrain sem segredos.
4. Rodar Graphify status/update conforme regra do repo; se rebuild for necessario, usar contencao de CPU.

A Phase 32 so fecha quando o canal `OpenAI - Codex` tiver credencial propria do Router com refresh-token, UI correta, erro operacional claro, docs/guardas atualizados, build/test/smoke verde, commit e push feitos.
