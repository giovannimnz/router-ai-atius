<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PostgreSQL live → arquivo de backup | Dados de producao saem do runtime para artefato protegido fora do Git. |
| backup → PostgreSQL k3s | SQL nao confiavel para o target atravessa psql. |
| PVC → PV local-path | Lifecycle do claim pode apagar dados se reclaim policy permanecer Delete. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-29-05 | Spoofing | origem PostgreSQL | high | mitigate | confirmar identidade do DB e invariantes antes do pg_dump |
| T-29-06 | Tampering | dump/restore | high | mitigate | SHA-256, target limpo, ON_ERROR_STOP e invariantes pos-restore |
| T-29-07 | Information Disclosure | backup/evidencia | high | mitigate | mode restritivo, fora do Git e evidencia sanitizada sem DSN/Secret |
| T-29-08 | Denial of Service | PV local-path | high | mitigate | patch e read-back de Retain antes do restore; proibir delete no rollback |
| T-29-SC | Tampering | imagens Postgres | high | mitigate | usar imagem previamente auditada e identidade imutavel; sem installs |
</threat_model>
