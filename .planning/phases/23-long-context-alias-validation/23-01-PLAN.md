# Plan 23-01 — Progressive 1M Long-Context Alias Validation

## Objective

Create and validate a safe test harness for base models `gpt-5.5`/`gpt-5.4` and aliases `gpt-5.5-1m`/`gpt-5.4-1m`. The primary live check is sending `POST /v1/chat/completions` with the requested model selected in `model`. The harness must prove base-model normal request acceptance, base-model 1M rejection, alias request acceptance, alias routing readiness, streaming compatibility, progressive long-context acceptance, usage return, optional catalog exposure/pricing, and reasoning over anchors spread across large contexts up to approximately 1M tokens.

## Tasks

1. Build a progressive validation script in `scripts/test-long-context-aliases.sh`.
2. Add non-streaming `POST /v1/chat/completions` validation for base models and aliases.
3. Add streaming `POST /v1/chat/completions` validation for base models and aliases.
4. Add optional `/v1/models` preflight validation for alias fields and pricing.
5. Add deterministic reasoning prompts with start/middle/end anchors and a verification code.
6. Add base-model limit-guard semantics: 1M rejection is pass for `gpt-5.5`/`gpt-5.4`; 1M acceptance is fail.
7. Add alias long-context semantics: 1M acceptance is pass for `gpt-5.5-1m`/`gpt-5.4-1m`; rejection is fail.
8. Add step gates:
   - manual confirmation for `250000+`;
   - separate `ENABLE_1M=YES_I_ACCEPT_COSTS` gate for `1000000`.
9. Write JSONL evidence to `logs/long-context-aliases/`.
10. Add static tests that verify safety properties of the script.
11. Run only local/static tests now; leave live expensive tests pending operator approval.

## Acceptance Criteria

- The script supports `MODEL=all`, `MODEL=aliases`, `MODEL=base`, and each individual model.
- `MODEL=all` runs `gpt-5.5`, `gpt-5.5-1m`, `gpt-5.4`, and `gpt-5.4-1m`.
- Default sizes include `small 10000 50000 100000 250000 300000 500000 750000 1000000`.
- By default, the script sends `/v1/chat/completions` requests and skips catalog preflight unless `SKIP_PREFLIGHT=0` is set.
- `BASE_EXPECT_REJECT_FROM=1000000` marks 1M requests for base models as expected rejection tests.
- The 1M step cannot run unless `ENABLE_1M=YES_I_ACCEPT_COSTS` is set and the typed confirmation matches the model and size.
- Logs include `http_status`, `duration_ms`, `usage`, `error`, `passed`, and response excerpts without printing tokens.
- Static tests pass with `python3 scripts/test_long_context_aliases_static_test.py`.
- Syntax validation passes with `bash -n scripts/test-long-context-aliases.sh`.

## Validation Commands

```bash
bash -n scripts/test-long-context-aliases.sh
python3 scripts/test_long_context_aliases_static_test.py
```

Live validation, after deploying the alias pricing code:

```bash
export ROUTER_BASE_URL="https://router.atius.com.br"
export ROUTER_TEST_KEY="COLOQUE_UM_TOKEN_DE_TESTE_AQUI"

MODEL=all \
SIZES="small 10000 50000 100000 250000 300000 500000 750000" \
scripts/test-long-context-aliases.sh

Base-model original-limit check and alias 1M check:

```bash
MODEL=all \
SIZES="1000000" \
ENABLE_1M=YES_I_ACCEPT_COSTS \
scripts/test-long-context-aliases.sh
```

Optional catalog/pricing preflight:

```bash
MODEL=all \
SIZES="" \
RUN_STREAM_SMOKE=0 \
SKIP_PREFLIGHT=0 \
scripts/test-long-context-aliases.sh
```

Alias-only 1M chat-completions validation:

```bash
MODEL=aliases \
SIZES="1000000" \
ENABLE_1M=YES_I_ACCEPT_COSTS \
scripts/test-long-context-aliases.sh
```

## Rollback

- The script is additive and only sends requests when explicitly run.
- Remove or restore `scripts/test-long-context-aliases.sh` from the backup if needed.
- Delete local evidence under `logs/long-context-aliases/` only if the operator explicitly requests cleanup.
