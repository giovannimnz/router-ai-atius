<objective>
After canonical remote preservation and mainline reconciliation are done, remove stale local worktrees/branches and prune redundant remote PT branches.

The outcome should be one authoritative fork mainline and one authoritative remote PT handoff branch.
</objective>

## Deliverables

- recreated clean local `main`
- local stale worktrees removed
- local stale branches removed
- remote redundant PT branches removed or explicitly archived
- documented final branch policy

## Tasks

1. Delete the stale local `main` worktree and recreate it clean from `origin/main`.
2. Remove local stale branches/worktrees:
   - `feat/pt-native`
   - `feat/brazilian-portuguese-localization`
   - `fix/sync-upstream-tag-fetch`
   - optionally local `feat/phase21-pt-native-upstream` after remote preservation
3. Remove or archive redundant remote PT branches:
   - `origin/feat/pt-native`
   - likely `origin/feat/pt-native-i18n-clean`
4. Confirm the final branch policy in repo docs.

## Acceptance

- local environment has no stale worktree ambiguity
- remote preserves only the branches with real future value
- operators know which branch to use for fork mainline and which branch to use for Phase 21 upstream handoff

## Verification

- `git worktree list`
- `git branch -vv`
- `git branch -r`
- `git ls-remote --heads origin`
- doc check for `docs/BRANCH-HYGIENE-PT-NATIVE.md`
