# Phase 32-02 PLAN — UI Codex-specific credential panel

## Objetivo

Remover do editor Codex as superficies genericas de API/base URL/API key e adicionar painel OAuth com `Atualizar credencial` e `Regenerar credencial`.

## Escopo

- `web/default/src/features/channels/api.ts`
- `web/default/src/features/channels/types.ts`
- `web/default/src/features/channels/constants.ts`
- `web/default/src/features/channels/components/drawers/channel-mutate-drawer.tsx`
- `web/default/src/i18n/locales/pt.json`

## Tarefas

1. Adicionar client APIs para credential metadata, regenerate start, regenerate complete e probe.
2. Adicionar tipo/DTO frontend para metadata sanitizada.
3. Criar condicao `isCodexChannel` e excluir type `57` da renderizacao de `Base URL`.
4. Nao renderizar `FormField name='key'` generico quando `currentType === 57`.
5. Renderizar card `Credencial OAuth Codex` com estado:
   - autenticado
   - account/email
   - expiration
   - has_refresh_token
   - last_refresh
   - last_probe
   - last_upstream_auth_error
   - requires_regeneration
6. Adicionar botao `Atualizar credencial` chamando refresh.
7. Adicionar botao `Regenerar credencial` com modal de start/complete OAuth.
8. Incluir copy PT-BR completo no `pt.json`.
9. Corrigir label de type `57` para `OpenAI - Codex` se ainda estiver divergente.
10. Adicionar teste ou smoke DOM para garantir que os textos genericos nao aparecem em type `57`.

## Validacao da Wave

Comandos obrigatorios:

```bash
git diff --check -- web/default/src/features/channels web/default/src/i18n/locales/pt.json
./scripts/podman-admin.sh profile-run -- bash -lc 'cd web/default && bun run i18n:sync && bun run typecheck && bun run build'
```

Validacoes logicas:

- Type `57` nao mostra `Base URL`.
- Type `57` nao mostra `API Key *`, `Current key`, `Reveal key` ou copy de key.
- Type `57` mostra `Atualizar credencial` e `Regenerar credencial`.
- Modal de regeneration nao exibe token, apenas authorize URL/callback input e status.

## Criterio de Conclusao

Operador consegue entender e executar refresh/regeneration sem tocar em base URL ou key bruta para Codex.
