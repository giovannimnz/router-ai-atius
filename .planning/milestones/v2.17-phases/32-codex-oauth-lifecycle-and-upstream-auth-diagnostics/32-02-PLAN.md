<interfaces>
Backend contract created by 32-01 and consumed here:
- `GET /api/channel/:id/codex/credential` returns `{ success, message?, data? }` with sanitized metadata fields: `channel_id`, `channel_type`, `channel_name`, `authenticated`, `has_refresh_token`, `expires_at`, `last_refresh`, `account_id`, `email`, `last_probe_at`, `last_probe_status`, `last_upstream_status`, `last_upstream_auth_error`, `requires_regeneration`, and `regeneration_reason`.
- `POST /api/channel/:id/codex/refresh` refreshes only with an existing refresh token and returns sanitized metadata or a regeneration-required error.
- `POST /api/channel/:id/codex/probe` explicitly probes upstream auth and returns sanitized metadata.
- `POST /api/channel/:id/codex/regenerate/start` returns an authorize URL for Router-owned Authorization Code + PKCE.
- `POST /api/channel/:id/codex/regenerate/complete` accepts an input string containing either the final callback URL or `code#state`, completes the token exchange, and returns sanitized metadata.

Existing frontend surfaces to reuse:
- API calls use `api` from `@/lib/api` and `channelActionConfig()` from `web/default/src/features/channels/api.ts`.
- Drawer conditional rendering is driven by `currentType = form.watch('type')`.
- Channel type labels come from `CHANNEL_TYPES` and `CHANNEL_TYPE_OPTIONS` in `web/default/src/features/channels/constants.ts`.
- The drawer already uses `Alert`, `Badge`, `Button`, `Input`, `Textarea`, `Loader2`, `RefreshCw`, `toast`, `useQuery`, and `useQueryClient`; preserve those patterns.
- Current code already blocks batch/multi-key modes for type 57; keep that behavior.
</interfaces>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Admin browser -> Router admin API | Authenticated admin actions trigger sensitive Codex refresh/probe/regeneration calls. |
| Router admin API -> React Query/UI state | Sanitized metadata crosses into browser memory and rendered UI. |
| Operator -> regeneration modal input | Callback URL or `code#state` is entered manually and must remain transient. |
| Browser -> external OpenAI authorization page | UI opens an external authorize URL and returns through manual callback completion. |
| Build/test tooling -> bundled assets | Frontend bundle and smoke output must not contain token material or persisted callback data. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-32-02-S1 | Spoofing | Regeneration modal | high | mitigate | Show channel id/name context from sanitized metadata, require saved channel id, call only 32-01 channel-scoped endpoints, and rely on backend PKCE state validation. |
| T-32-02-T1 | Tampering | Type 57 drawer conditional rendering | medium | mitigate | Gate all generic credential blocks behind non-Codex conditions and add smoke coverage that fails if generic controls reappear for Codex. |
| T-32-02-R1 | Repudiation | Refresh/probe/regenerate UI actions | medium | mitigate | Show sanitized success/error toasts and visible status fields from backend metadata; do not invent local success when the backend returns failure. |
| T-32-02-I1 | Information Disclosure | Browser UI, DOM, clipboard, logs, bundle | high | mitigate | Model only sanitized metadata, never render token fields, never reveal/copy saved Codex key, never store callback/token values, clear transient modal input, and assert fake token values are absent from smoke output. |
| T-32-02-D1 | Denial of Service | Probe/refresh buttons | medium | mitigate | Keep probe and refresh as explicit operator actions with loading/disabled states and no automatic upstream probe on drawer load. |
| T-32-02-E1 | Elevation of Privilege | Sensitive channel actions | high | mitigate | Preserve existing sensitive-write permission gating; disable refresh/regeneration when `sensitiveLocked` or no saved channel id exists. |
| T-32-02-SC | Tampering | Package installs | high | mitigate | No npm, Bun, pip, cargo, or Go dependency install is in scope; use existing React, React DOM, Bun, and node:test capabilities only. |
