## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Admin UI/API client -> `/api/channel/:id/codex/*` | Authenticated admin input can trigger sensitive Codex credential operations. |
| Router backend -> Codex upstream OAuth/API | Router sends OAuth credentials to upstream and receives untrusted upstream error bodies. |
| Router backend -> database `channels.key` and `channels.setting` | Secret token JSON and non-secret health metadata are stored in the same channel row. |
| Relay client -> `/v1/*` -> upstream Codex | Client requests can surface upstream auth failures through relay responses. |
| Test harness -> local fixtures | Negative auth tests must remain isolated from live channel 5 and production secrets. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-32-01-S1 | Spoofing | Codex admin routes | high | mitigate | Require existing channel permissions, validate channel id, validate channel type 57, and keep regenerate complete bound to cached PKCE state/verifier. |
| T-32-01-T1 | Tampering | `channel.setting` health writes | high | mitigate | Merge `codex_credential_health` into existing `dto.ChannelSettings` and add regression coverage that unrelated settings survive updates. |
| T-32-01-R1 | Repudiation | Refresh/probe/regenerate failures | medium | mitigate | Record sanitized upstream status/code and regeneration reason in health metadata while avoiding secrets in logs or responses. |
| T-32-01-I1 | Information Disclosure | Metadata DTO, logs, test output | high | mitigate | Never return token fields, mask sensitive error details, bound upstream body previews, and add tests asserting fake token material is absent from serialized metadata. |
| T-32-01-D1 | Denial of Service | Probe and catalog discovery | medium | mitigate | Keep probe explicit, bounded by context timeout, and absent from ordinary metadata reads; keep tests mocked. |
| T-32-01-E1 | Elevation of Privilege | Regenerate complete and refresh | high | mitigate | Use sensitive channel-write permission for refresh/regenerate, read-only permission for metadata, and channel type validation before credential writes. |
| T-32-01-SC | Tampering | Package installs | low | accept | No npm, pip, cargo, or Go dependency install is in scope; executor must not add dependencies for this plan. |
</threat_model>
