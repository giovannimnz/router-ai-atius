# Phase 02: pt Fumadocs i18n — Context

**Gathered:** 2026-06-05
**Status:** Ready for planning
**Milestone:** v2.12 — pt Native i18n Integration (continuation)

## Phase Boundary

Add Portuguese locale (`pt`) to the Atius-branded documentation site (Next.js 16 + Fumadocs Core 16) that serves `https://router.atius.com.br/{en,zh,ja}/docs/`. Follow the EXACT same pattern used by the existing `en` and `ja` locales — directory-based content (`content/docs/{locale}/`), config-based locale list (`src/lib/i18n.ts`), and the existing translation script (`scripts/translate-docs.ts`).

This phase delivers:
- `/pt/docs/` route resolving with PT-BR content
- ~302 additional MDX files translated from en to pt-BR (apps, skills, support, business, console, feature-guide, all api/* endpoints)
- `rel="alternate" hrefLang="pt"` link auto-generated by Fumadocs
- Zero custom routing — only registration points in the existing native infrastructure

## Critical Architecture Insight

The docs site is **NOT** the `integration/docs/` directory in this repo. The actual app is:

- **Fork**: `giovannimnz/new-api-docs-v1` (Atius-branded) — lives in `/home/ubuntu/GitHub/refs/new-api-docs-v1/` (or a fresh local clone)
- **Build context**: the fork repo
- **Docker image**: `localhost/router-ai-atius-docs:local` (running on `:3003`)
- **Source config**: `refs/new-api-docs-v1/source.config.ts` (compiled to `integration/docs/.source/index.ts` for cross-repo type imports)

**The fork-sync project** at `/home/ubuntu/fork-sync/projects/atius-router-docs/` already has:
- `sync.yaml` with `protected_paths` declaring `content/docs/pt/**` as Atius-owned
- `pt-content/docs/pt/` with 11 PT-BR MDX files already translated (index, installation, guide, api)
- `Dockerfile.template` with Atius branding + `next start` command
- `atius-router-docs-rebrand.sh` post-merge script (replaces QuantumNous branding)
- `fork-sync-docs.sh` (entrypoint for fork-sync operations)

The 11 PT files in `pt-content/` are **translated seed content** that gets merged into the fork repo's `content/docs/pt/` during fork-sync. Phase 02 must:

1. **Add `pt` to the fork repo's i18n config** (`src/lib/i18n.ts` + `next.config.mjs`)
2. **Translate the remaining ~302 EN files** to PT-BR using `scripts/translate-docs.ts` (adapt LANGUAGES)
3. **Place all 313+ PT files** at `content/docs/pt/` in the fork repo
4. **Update fork-sync `sync.yaml`** so the PT files become part of the protected content
5. **Rebuild the Docker image** with PT locale and redeploy

## Implementation Decisions

### D-01: Translation Scope
- **All 313 EN files** — full mirror of `en/` structure
- Sections covered: guide, installation, api, apps, skills, support, business
- Already have 11 files (index, installation, guide, api subdirs) — 302 more needed
- `zh/` has 355 files (en + `legal/` extra) — `pt/` will mirror `en/` (313 files), no legal section unless explicitly added later

### D-02: Translation Method
- **Adapt `translate-docs.ts` existing** — add `pt` to `LANGUAGES` object
- Current LANGUAGES: `en: { ... }, ja: { ... }` — both translated FROM `zh`
- Add: `pt: { name: 'Portuguese', nativeName: '葡萄牙语', dir: 'pt' }`
- Use existing batch infrastructure: workers (MAX_WORKERS=3), retries, OpenAI-compatible API (gemini-2.5-flash)
- Source language: `en` (pt will be translated FROM en, not from zh — require source override)
- Update `GLOSSARY` to include en→pt mappings (no Chinese terms in source)
- Run from the fork repo (not from router-ai-atius)

### D-03: pt Meta Structure
- **Same as en** — title: "Documentação", pages: [guide, installation, api, apps, skills, support, business]
- Already correct in `pt-content/docs/pt/meta.json`
- No `legal/` section (zh-only)
- Standard sidebar structure identical to en/ja
- No custom section additions

### D-04: Validation
- **typecheck + build + Playwright/Chromium browser validation**
- `bun run typecheck` (or `tsc -b`) passes
- `next build` produces `.next/` with `/pt/docs/` route
- Playwright/Chromium validates:
  - `/pt/docs/` returns 200, renders PT-BR content
  - `/pt/docs/skills/` returns 200, renders PT-BR
  - `rel="alternate" hrefLang="pt"` present in `<head>`

## Existing Code Insights

### Reusable Assets
- `defineI18n()` from `fumadocs-core/i18n` — declarative locale config
- `createI18nMiddleware()` — auto-generates middleware from i18n config
- `translate-docs.ts` — batch translate with workers, retries, incremental mode
- `.github/workflows/translate.yml` — CI auto-translate on push to main
- 11 existing PT files in `pt-content/docs/pt/` — proven translation patterns

### Established Patterns
- Each locale is a directory: `content/docs/{locale}/`
- Each locale has `index.mdx` (root page) + `meta.json` (sidebar order)
- Same content structure across locales (same filenames, same MDX components)
- `rel="alternate" hrefLang="..."` auto-generated by Fumadocs
- Headers regex `:lang(en|zh|ja)` must include all active locales
- Fork-sync uses `protected_globs` to keep Atius-specific files (like PT content) safe from upstream merges
- Docker rebuild required to deploy locale changes (no runtime i18n switching)

### Integration Points
