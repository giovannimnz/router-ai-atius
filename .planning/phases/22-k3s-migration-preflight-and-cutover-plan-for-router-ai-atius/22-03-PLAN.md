<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Podman DB -> backup artifact | Sensitive data leaves running DB into backup directory. |
| Backup artifact -> k3s Postgres | Restore rehearsal can create persistent data in cluster storage. |
| Shadow Service -> smoke clients | Test traffic exercises provider and embeddings paths. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-22-03-01 | Information Disclosure | backups | high | mitigate | Store under backups, keep out of build context, do not print secrets, record path only. |
| T-22-03-02 | Denial of Service | shadow apply | high | mitigate | Require opt-in env flag and do not touch Apache/public traffic. |
| T-22-03-03 | Tampering | provider/channel DB | high | mitigate | Backup channel/token state before restore/base URL changes. |
| T-22-03-04 | Reliability | local-path storage | high | mitigate | Restore rehearsal and explicit non-HA warning before cutover. |
</threat_model>
