<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| Client JSON -> DTO | Untrusted embedding input is parsed into request structs. |
| DTO -> relay | Relay derives numeric metadata from parsed input. |
| Relay -> governor | Metadata, workload header, and upstream status influence adaptive throttling. |
| Relay -> upstream provider | Existing provider adaptors receive the unchanged converted request. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|---|---|---|---|---|---|
| T-20-04-01 | Information Disclosure | DTO stats helper / relay wiring | high | mitigate | Helper returns only numeric stats; relay passes no raw inputs, request bodies, Authorization values, or provider keys to the governor. |
| T-20-04-02 | Tampering | Workload header | medium | mitigate | Header affects classification but remains bounded by governor queue limits, batch cap, and request-derived fallback. |
| T-20-04-03 | Denial of Service | Relay status propagation | medium | mitigate | Transport failures stay pressure-class; ordinary upstream client statuses rely on governor classifier so bad requests do not throttle all clients. |
| T-20-04-04 | Spoofing | Public model alias mapping | low | mitigate | Relay preserves public model name for governor scope before upstream model mapping, avoiding accidental bypass by mapped provider names. |
| T-20-04-SC | Tampering | Package supply chain | low | accept | No package install is planned; implementation uses existing Go stdlib and dependencies. |
</threat_model>
